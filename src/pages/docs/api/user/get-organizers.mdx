# `getOrganizers`

This documentation covers the `getOrganizers` function, which is responsible for retrieving all user records with the role of "organizer" from the database in the application.

## Function: `getOrganizers`

### Description

The `getOrganizers` function is an asynchronous function that retrieves all user records from the MongoDB database where the user's role is "organizer".

### Parameters

- None

### Return Value

- Returns a JSON stringified array of user records with the role of "organizer" if successful.
- Throws an error if there is any issue during the retrieval process.

### Example Usage

```javascript
export async function getOrganizers() {
  try {
    await connectToDatabase();

    const organizers = await User.find({ role: "organizer" });

    return JSON.parse(JSON.stringify(organizers));
  } catch (error) {
    handleError(error);
  }
}
```

### Error Handling

- The function uses a try-catch block to handle errors.
- Any errors encountered during the retrieval process are passed to the `handleError` function for handling.

### Dependencies

- `connectToDatabase`: A function to connect to the MongoDB database.
- `User`: A MongoDB model representing the user.
- `handleError`: A function to handle errors.

### Notes

- This function assumes that the necessary MongoDB model (`User`) and utility functions (`connectToDatabase`, `handleError`) are properly defined and imported in your application.

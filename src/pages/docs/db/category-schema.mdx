# Category Schema Documentation

This document provides a detailed description of the `Category` schema used in our application. The schema is defined using Mongoose, a MongoDB object modeling tool designed to work in an asynchronous environment.

## Category Interface

The `ICategory` interface extends the Mongoose `Document` and defines the structure of a Category object.

```typescript
import { Document } from "mongoose";

export interface ICategory extends Document {
  _id: string;
  name: string;
}
```

## Category Schema

The `CategorySchema` defines the structure and constraints of the Category documents stored in the MongoDB collection.

```typescript
import { Schema, model, models } from "mongoose";

const CategorySchema = new Schema({
  name: { type: String, required: true, unique: true },
});
```

## Category Model

The `Category` model is created using the `CategorySchema`. It is either retrieved from the existing models or created if it does not exist.

```typescript
const Category = models.Category || model("Category", CategorySchema);

export default Category;
```

### Field Descriptions

- **\_id**: The unique identifier for the category (automatically generated by MongoDB).
- **name**: The name of the category (required, unique).

This schema ensures that all necessary fields are validated and appropriately constrained, providing a robust structure for storing category-related data in the database.

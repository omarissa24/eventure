# Task Schema Documentation

This document provides a detailed description of the `Task` schema used in our application. The schema is defined using Mongoose, a MongoDB object modeling tool designed to work in an asynchronous environment.

## Task Interface

The `ITask` interface extends the Mongoose `Document` and defines the structure of a Task object.

```typescript
import { Document } from "mongoose";

export interface ITask extends Document {
  _id: string;
  event: { _id: string; title: string };
  creator: { _id: string; firstName: string; lastName: string };
  title: string;
  description: string;
  status: string;
  deadline: Date;
  assignee: { _id: string; firstName: string; lastName: string };
}
```

## Task Schema

The `TaskSchema` defines the structure and constraints of the Task documents stored in the MongoDB collection.

```typescript
import { Schema, model, models } from "mongoose";

const TaskSchema = new Schema({
  event: { type: Schema.Types.ObjectId, ref: "Event" },
  creator: { type: Schema.Types.ObjectId, ref: "User" },
  title: { type: String, required: true },
  description: { type: String },
  status: { type: String, required: true },
  deadline: { type: Date, required: true },
  assignee: { type: Schema.Types.ObjectId, ref: "User" },
});
```

## Task Model

The `Task` model is created using the `TaskSchema`. It is either retrieved from the existing models or created if it does not exist.

```typescript
const Task = models.Task || model("Task", TaskSchema);

export default Task;
```

### Field Descriptions

- **\_id**: The unique identifier for the task (automatically generated by MongoDB).
- **event**: The event associated with the task, referencing the `Event` model.
- **creator**: The creator of the task, referencing the `User` model.
- **title**: The title of the task (required).
- **description**: A detailed description of the task (optional).
- **status**: The current status of the task (required).
- **deadline**: The deadline for the task completion (required).
- **assignee**: The user assigned to the task, referencing the `User` model.

This schema ensures that all necessary fields are validated and appropriately referenced, providing a robust structure for storing task-related data in the database.
